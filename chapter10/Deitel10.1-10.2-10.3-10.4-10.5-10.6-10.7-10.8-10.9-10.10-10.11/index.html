10.1 Preencha as lacunas em cada uma das seguintes afirmações:

a) Se uma classe contiver pelo menos um método abstrato, ela será uma classe abstrata.

b) As classes a partir das quais os objetos podem ser instanciados são chamadas concretas.

c) Polimorfismo envolve a utilização de uma variável de superclasse para invocar métodos nos objetos de superclasse e de subclasse, permitindo que você “programe no geral”.

d) Os métodos que não são métodos de interface e que não fornecem implementações devem ser declarados com a palavra-chave abstract.

e) Fazer uma coerção em uma referência armazenada em uma variável da superclasse para um tipo de subclasse é chamado downcasting.

10.2 Determine se cada uma das instruções a seguir é verdadeira ou falsa. Se falsa, explique por quê.

a) Todos os métodos em uma classe abstract devem ser declarados como métodos abstract. (F) Uma classe abstrata pode incluir métodos com implementações.

b) Não é permitido invocar um método “somente de subclasse” por meio de uma variável de subclasse.(F) Se a variável for da subclasse e a referência também, é possível invocar o método da subclasse.

c) Se uma superclasse declarar um método como abstract, uma subclasse deverá implementar esse método.(F) Se a subclasse for abstract também, não será necessário implementar o método. Sendo assim, apenas uma subclasse concreta deve implementar o método.

d) Um objeto de uma classe que implementa uma interface pode ser pensado como um objeto desse tipo de interface. (V)

10.3 (Interfaces Java SE 8) Preencha os espaços em branco em cada uma das seguintes instruções:

a) No Java SE 8, uma interface pode declarar métodos default, isto é, métodos public com implementações concretas que especificam como uma operação deve ser realizada.

b) A partir do Java SE 8, as interfaces agora podem incluir métodos auxiliares public static.

c) A partir do Java SE 8, qualquer interface que contém um único método é conhecida como interface funcional.

10.4 Como o polimorfismo permite programar “no geral” em vez de “no específico”? Discuta as vantagens cruciais da programação “no geral”.

Com o polimorfismo é possível criar sub classes que herdam da super classe e atribuir intefaces. Com isso, caso tenhamos objetos de diferentes sub classes mas que herdam a mesma classe, podemos atribui-los a uma variável da super classe e conseguir manipular todos. Além disso, com as interfaces conseguimos fazer a mesma coisa. Caso tenhamos duas classes totalmente diferentes, mas com a mesma interface, é possível atribuir a variável da interface e a referência das classes, conseguindo assim, manipula-las.

10.5 O que são métodos abstratos? Descreva as circunstâncias em que um método abstrato seria apropriado.

Métodos abstratos são métodos que não possuem implementação, sendo assim, a classe concreta deve implementa-lo. São apropriados quando temos uma classe genérica e diversas classes concretas diferentes, mas com a mesma ação.

10.6 Como o polimorfismo promove extensibilidade?

Promove a extensibilidade porque é possível adicionar novas subclasses com o comportamento da super classe.

10.7 Discuta três maneiras como você pode atribuir referências de superclasse e de subclasse a variáveis de superclasse e a tipos de subclasse.

superclasse - superclasse | subclasse - subclasse | superclasse - subclasse.

10.8 Compare e contraste classes abstratas e interfaces. Por que você utilizaria uma classe abstrata? Por que você utilizaria uma interface?

Classes abstratas podem ter variáveis, métodos comuns e métodos abstratos, mas não podem ser instanciadas. Interfaces a partir do Java SE 8 podem definir constantes, métodos estáticos, métodos default e métodos public abstratos. Utilizamos classes abstratas quando queremos que as subclasses tenham um padrão para ser usado. Utilizamos interfaces quando queremos firmar um contrato, sendo que, diversas classes que não tem relação uma com a outra podem utilizar a inteface.

10.9 (Interfaces Java SE 8) Explique como métodos default permitem adicionar novos métodos a uma interface existente sem violar as classes que implementaram a interface original.

O método default já vem por padrão nas classes que implementam a interface, ou seja, não é necessário implementar uma lógica nele. No entanto, é possível sobrescreve-lo. Com isso, é possível implementar métodos default sem interferir nas classes que já implementam a interface.

10.10 (Interfaces Java SE 8) O que é uma interface funcional?

Uma interface funcional é uma interface que possui apenas um método

10.11 (Interfaces Java SE 8) Por que é útil ser capaz de adicionar métodos static a interfaces?

Porque agora podemos adicionar métodos auxiliares na própria interface em vez de declara-los em classes diferentes.
