3.5 (Palavra-chave new) Qual é o objetivo da palavra-chave new? Explique o que acontece quando você a utiliza.

O objetivo da palavra new é criar uma referência de memória. Além disso, inicializar as variavéis de instâncias que a classe possue.

3.6 (Construtores padrão) O que é um construtor padrão? Como as variáveis de instância de um objeto são inicializadas se uma classe tiver somente um construtor padrão?

Um construtor padrão serve para criar um objeto de determinada classe. As variáveis de instâncias são inicializadas com 0 se forem do tipo primitivo e null se forem do tipo referência.

3.7 (Variáveis de instância) Explique o propósito de uma variável de instância.

O próposito das variáveis de instância é guarda valores para determinados objetos. Quando criamos uma classe, podemos declarar variáveis de instâncias e instânciar objetos, sendo que, os objetos terão cópias das variáveis de instâncias.

3.8 (Usando classes sem importá-las) A maioria das classes precisa ser importada antes de ser usada em um aplicativo. Por que cada aplicativo pode utilizar as classes System e String sem importá-las antes?

O java possue o pacote java.lang que é importado implicitamentes em todas as classes. Sendo que, as classes System e String fazem parte desse pacote.

3.9 (Usando uma classe sem importá-la) Explique como um programa pode usar a classe Scanner sem importá-la.

O programa pode utilizar o nome de classe totalmente qualificado. Exemplo: java.util.Scanner input = new java.util.Scanner(System.in); Sendo assim, não vai ser necessário o import da classe Scanner.

3.10 (Métodos set e get) Explique por que uma classe pode fornecer um método set e um método get para uma variável de instância

Para maior seguraça das variáveis de instâncias, podemos utilizar o modificador de acesso private. Sendo assim, podemos limitar se queremos expor essa variável. Para conseguirmos acessar ou modificar essas variáveis utilizamos o set e get.
