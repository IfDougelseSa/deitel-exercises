1.7) Explique as duas fases de compilação de programas Java.

A primeira fase de compilação do programa java, ocorre quando utilizamos o comando javac. Ao utilizarmos esse comando, o código fonte .java é compilado para bytecodes .class. A segunda fase ocorre quando chamamamos o programa java com o comando java. Na fase de execução esse comando vai pegar o bytecodes e compilar o código para a linguagem de máquina do SO que está chamando.

1.8) Um dos objetos mais comuns do mundo é um relógio de pulso. Discuta como cada um dos seguintes termos e conceitos se aplicam à noção de um relógio: objeto, atributos, comportamentos, classe, herança (considere, por exemplo, o alarme dele), modelagem, mensagens, encapsulamento, interface e ocultamento de informações.

Objeto) Podemos ter uma classe Relógio e criar diversos relógios instanciando essa classe. Exemplo: Relogio relogio1 = new Relogio().

Atributos) A classe relógio pode ter diversos atributos como: formato do relógio, tipo do relógio, horário atual, etc...

Comportamentos) Podemos ter métodos para o relógio. Exemplo: alterar horário, alterar brilho, etc...

Classe) Podemos criar uma classe para o relógio, como se fosse um modelo. A partir da classe, podemos criar vários relógios com instâncias.

Herança) Podemos criar outra classe que tenha mais atributos ou metódos que um relógio comum. Por exemplo, um relógio que tenha alarme. Podemos aproveitar tudo do relógio comum e apenas adicionar o alarme na nova classe extendendo a classe pai.

Modelagem) Podemos usar a UML para modelar as classes.

Mensagem) O envio de uma mensagem a um objeto faz o método correspondente ser executado.

Encapsulamento) Podemos deixar os atributos como privados, encapsulando o mesmo. Assim, para acessar ou modificar os atributos, precisamos criar gets e setters.

Interface) Podemos criar interfaces para determidas classes seguirem regras pré-definidas.

Ocultamento de informações) Podemos deixar os atributos privado, assim ocultando informações para outras classes.
